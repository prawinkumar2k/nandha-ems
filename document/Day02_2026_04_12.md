# Day 2: Dynamic Question Engine (2026-04-12)

## Objective
Transform the static exam system into a polymorphic Engine capable of supporting multiple question formats beyond simple MCQs.

## Tasks Completed
1. **Schema Polymorphism Implementation**:
   - Refactored the `Exam` and `QuestionBank` MongoDB schemas to support a robust `type`-based structure.
   - Added support for new question types: `Coding` (with test cases), `Text` (long-form), `Math` (LaTeX), and `File Upload`.
2. **React Dynamic Form Factory**:
   - Built a specialized frontend factory that dynamically renders the correct input interface based on the question type.
   - Integrated a code editor component for programming assessments.
3. **Assessment Logic**:
   - Updated the submission handlers to process diverse answer payloads (e.g., code strings vs. MCQ selections).

## Outcome
Shifted the platform from a simple quiz tool to a professional assessment engine suitable for technical and academic variety.
